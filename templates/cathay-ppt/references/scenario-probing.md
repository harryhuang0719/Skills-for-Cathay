# 7 条判断场景 — IB/PE Deck 专用

> 蒸馏自 aham-ppt 24 场景，只保留对 IB/PE 场景跨项目适用的判断。
>
> **"答案"跨项目不一样，但"提问方式"跨项目复用。**
>
> 这些场景的价值是**防止关键节点漏判**。每条场景都是曾经在某个项目踩过的坑。

---

## S1 · 规模语境校准

**触发点**：接到做 deck 需求的第一刻（阶段 1 规范加载前）

**必问**：
1. 篇幅目标？3 页 teaser / 10 页 IC memo quick take / 30 页正式 IC / 50 页路演稿？
2. 受众？GP / LP / 投委会 / 创始人 / 外部客户？
3. 交付场景？邮件发送 / 现场路演 / 大屏投影 / 打印版？

**为什么重要**：
- 3 页 teaser 全是结论，不讲过程
- 30 页 IC 需要完整金字塔 + 尽调支撑
- 50 页路演稿要留"喘息页"（章节 divider / Q&A cushion）

**跳过代价**：做到一半用户说"太多了"或"太少了"全部返工。

---

## S2 · 受众心理动线建模

**触发点**：Ghost Deck 前（阶段 4）

**必问**：
- LP 看：第一眼看**回报** → 然后看**风险** → 最后看 team
- 合伙人 IC：第一眼看 **deal 逻辑** → 风险 → 退出路径 → team
- 创始人：第一眼看**条款** → 然后看 sponsor 增值 → Roadmap
- 外部客户 pitch：第一眼看**对我有什么用** → 然后才是你是谁

**为什么重要**：同一个 Sealien deck，给 LP 看和给创始人看，章节顺序完全不同。

**跳过代价**：章节顺序错位，LP 看前 5 页找不到回报，注意力已经走了。

---

## S3 · 第三章灵魂页取向

**触发点**：阶段 5 大纲规划，确定 deck 主章节结构时

**必问**：核心叙事章节（通常是第三章）是"**投资亮点**"、"**风险与对冲**"还是"**交易结构**"？

这个问题看起来简单，但它决定了整个 deck 的重心：
- 亮点优先 → 偏 pitch，推销感强
- 风险优先 → 偏 IC memo，尽调感强
- 结构优先 → 偏 term sheet 讨论稿，谈判感强

**为什么重要**：银行家/VC 的 IC 流程里，第三章放什么是灵魂。选错了整份 deck 气质就不对。

**跳过代价**：deck 发出去 LP 以为你还没尽调，或者合伙人以为你在推销。

---

## S4 · 数字诚实度校验

**触发点**：任何具体数字进入 deck 时（标题 / 表格 / 图表）

**必问** — 每条数字：
1. **来源**：哪个文件、哪一页、哪个日期？
2. **类型**：历史真数 / 管理层预测 / 行业研报 / 我的模型测算？
3. **情景**：如果是预测，是 Base / Bull / Bear 哪一条？
4. **可验证性**：第三方能复核吗？（LP 大概率会复核）

**硬规则**：
- 历史数：注明时间段 + 来源（"2023 收入 1.2 亿（审计报告 p.15）"）
- 预测数：标注 scenario（"2028 收入 10 亿 (Base)"）
- 市场数据：注明研报出处 + 年份（"Frost & Sullivan 2024"）
- 禁止无来源数字，**如果源头拿不到就不用这个数字**，不要估

**为什么重要**：banker 最丢脸的场景是在 IC 被问"这个数从哪来"答不上来。

**跳过代价**：LP 抽查一个数字穿帮，整份 deck 可信度作废。

## S5 · 继承 vs 重构判断

**触发点**：拿到创始人 / 甲方 / 前一位分析师已有的 deck，被要求"改一下"

**必问**：是**继承**（用现有 deck 做增量改动）还是**重构**（用现有内容重新组织）？

判断标准：
- 现有 deck 叙事骨架对 → 继承 + 改内容即可
- 现有 deck 叙事骨架错（Ghost Deck 不成立） → 重构，不要打补丁

**为什么重要**：很多 deck 在"继承层"改来改去改 5 轮还不对，根因是骨架本来就错，应该一开始就重构。

**跳过代价**：3 天改 15 版，用户依然不满意，最后回来重做。

## S6 · SVG 视觉通道判断

**触发点**：阶段 7 逐页设计，决定每页走 python-pptx 还是 SVG→PNG 通道

**判断规则**：

| 页面类型 | 通道 | 原因 |
|---------|------|------|
| 封面 / 章节 divider | **SVG 可选**（视觉加分） | 成品图，本来就不需要编辑 |
| 商业模式图 / 产业链 mapping | **SVG 推荐** | 复杂视觉 python-pptx 难做好 |
| 复杂图表 / chart | **SVG 可选** | PPT 原生 chart 难看难调 |
| 封面上的大标题 | **SVG 可以** | 排版自由度需求 |
| 财务表格 / Comps / Returns | **禁止 SVG** | banker 需要可编辑 |
| 数字表 / KPI 卡 | **禁止 SVG** | 同上 |
| 正文 bullet 页 | **禁止 SVG** | 可编辑性优先 |

**为什么重要**：SVG→PNG 嵌入的页是"**图片**"，打开后不可编辑。对表格页这是不可接受的。

**跳过代价**：deck 发出去 banker 要改个 Comps 数字改不动，回来骂。

详细用法见 `lib/svg_embed.py`。

## S7 · 跨页一致性全扫描

**触发点**：阶段 8 QC 交付前最后一关

**扫描项**：
- [ ] **TAM / 市占率 / 收入**：三者是否自洽？公司 2030 收入 ÷ TAM 2030 = deck 里的市占率？
- [ ] **Round A pre-money / 总融资 / sponsor check**：三者是否匹配？
- [ ] **Returns**：IRR / MOIC 是否和 Exit Case 的 PE × NI × Ownership 倒算一致？
- [ ] **Exit Case**：2030 年的退出估值是否和 comps 暗示的 PE 区间一致？
- [ ] **毛利率 / 净利率**：如果预测说 2030 毛利 45%、净利 25%，是否能从 deck 其他页（SG&A / 税率）回推出来？
- [ ] **情景标记**：Base / Bull / Bear 在所有出现的地方是否用同一个定义？
- [ ] **历史数字**：每个历史数字在 deck 里只出现一次（或多次但完全一致）？

**为什么重要**：LP 和合伙人在 IC 上会同时翻第 10 页和第 30 页对数字，不一致当场就会被指出。

**跳过代价**：IC 当场尴尬。最严重的可能让 deal 被否。

## 场景增补流程

未来项目中发现新的关键场景：

1. 项目结束后单独记录一条 `SCENARIO_NEW_YYYYMMDD.md`，描述 Trigger / 必问 / 跳过代价
2. 冷思考 1 周后再决定是否整合进本文档（避免带入项目特殊性）
3. 整合时判断是否真的跨项目适用 — 只在特定行业出现的场景**不应**进本文档

**原则**：**不要边做项目边改本文档。** 本文档的价值是跨项目沉淀，不是当前项目日志。
